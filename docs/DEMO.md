# Weighted Routing Demo

## 1 分钟总结

```text
这个 MVP 已经完成了 Weighted Routing 的核心闭环。

第一，节点建模包含 name、weight、enabled。
第二，route() 只从 enabled=true 且 weight>0 的节点中选择。
第三，没有可用节点时会抛出明确异常。
第四，我用自动化测试覆盖了正常场景、disabled 节点、0 权重节点、空列表和所有节点不可用。
第五，对于 5:3:2 这种概率型逻辑，我用大量调用加容忍区间验证，而不是要求精确相等。

实现上，核心逻辑放在 Service，Controller 只负责接收请求和返回响应。
验证上，核心规则已经被单元测试和 Web 测试覆盖，行为是可运行、可测试、可解释的。
```

## 核心闭环

- 节点建模：`name`、`weight`、`enabled`
- 路由规则：只从 `enabled=true` 且 `weight>0` 的节点中选择
- 异常规则：没有可用节点时抛出明确异常
- 测试覆盖：正常场景、disabled 节点、0 权重节点、空列表、所有节点不可用
- 概率验证：对 5:3:2 这类逻辑使用大量调用和容忍区间，而不是精确相等

## 如果继续扩展到生产环境

5. 和限流、熔断、降级结合。
