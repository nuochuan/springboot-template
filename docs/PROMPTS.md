# 1. 开始任务时的第一条 Prompt

```text
请先阅读以下文件：

- AGENTS.md
- REQUIREMENTS.md
- TEST_CASES.md
- PLAN.md

请先不要写代码,你先做三件事：

1. 检查 REQUIREMENTS.md 是否清楚定义了目标、输入、输出、核心规则、边界情况和暂不实现内容。
2. 检查 TEST_CASES.md 是否覆盖正常场景、边界场景和异常场景。
3. 检查 PLAN.md 是否能支持一个可完成的 MVP 版本。

要求：
- 只指出缺失点和风险点。
- 不要创建 Java 文件。
- 不要修改代码。
- 不要添加数据库、Redis、MQ、登录、复杂前端。
```

```text

1. 建模步骤

4. DTO 设计
```

# 5. 基于文件实现 MVP

适用场景：REQUIREMENTS.md、TEST_CASES.md、PLAN.md 都已经准备好，正式让 AI 生成代码。

```text
请先读取以下文件：

- AGENTS.md
- REQUIREMENTS.md
- TEST_CASES.md
- PLAN.md

现在请基于这些文件实现当前任务的 Spring Boot + Java 8 + Gradle MVP。

实现要求：

1. 只实现 PLAN.md 中定义的内容。
2. 不要添加数据库。
3. 不要添加 Redis。
4. 不要添加 MQ。
5. 不要添加配置中心。
6. 不要添加登录权限。
7. 不要添加复杂前端。
8. 不要引入不必要的第三方依赖。
9. 核心业务逻辑必须放在 Service。
10. Controller 只负责参数接收、调用 Service、返回响应。
11. 使用 DTO 表达请求和响应。
12. 使用 JUnit 5 编写自动化测试。
13. 测试必须覆盖 TEST_CASES.md 中列出的场景。
14. 如果已有通用响应类和异常类，优先复用。
15. 不要修改 REQUIREMENTS.md、TEST_CASES.md、PLAN.md，除非我明确要求。

实现完成后，请输出：

1. 创建了哪些文件。
2. 修改了哪些文件。
3. 核心实现逻辑是什么。
```

```text

【测试名称】
在这里填写测试方法名。

【输入】
在这里填写输入参数。

【期望输出】
在这里填写期望结果。

【实际输出】
在这里填写实际结果。

【错误日志】
在这里粘贴关键错误日志。

请按以下步骤处理：

1. 先解释失败根因。
2. 判断是实现问题、测试问题，还是需求不清。
3. 给出最小修改方案。
4. 只修改必要文件。
5. 不要重写整个模块。
6. 不要修改 REQUIREMENTS.md、TEST_CASES.md、PLAN.md，除非明确是需求文件错误。
7. 修复后说明修改点和需要重新运行的测试命令。

请先输出根因分析，不要立即大范围改代码。
```
