# 需求说明

## 目标

实现一个年度工资薪金个人所得税计算器的 MVP 版本。

用户输入年度工资收入、年度专项扣除和年度其他扣除后，系统返回应纳税所得额、应纳税额、适用税率和速算扣除数。

## 任务描述

- 实现一个 50 分钟内可以完成的基础版本。
- 只计算年度工资薪金所得的个人所得税。
- 核心计算逻辑必须放在 Service 层，并且可以被单元测试覆盖。
- Controller 只负责接收参数、调用 Service 和包装响应。

## MVP 范围

本次只实现以下功能：

1. 根据输入金额计算年度个人所得税。
2. 校验必填输入和非法金额。
3. 返回统一的成功或失败响应。

本次不实现批量计算、历史记录、税收优惠、专项附加扣除明细拆分、多币种、多地区政策差异。

## 输入

接口或 Service 方法接收一个对象，字段如下：

| 字段名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| annualSalaryIncome | BigDecimal | 是 | 年度工资收入，单位：元 |
| annualSpecialDeduction | BigDecimal | 是 | 年度专项扣除，单位：元 |
| annualOtherDeduction | BigDecimal | 是 | 年度其他扣除，单位：元 |

输入约束：

1. 三个字段都不能为空。
2. 三个字段都必须是数字金额。
3. 三个字段都不能小于 0。
4. 金额允许为 0。
5. 金额最多按 2 位小数参与计算；超过 2 位小数时按非法输入处理。

## 输出

成功时返回以下数据：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| taxableIncome | BigDecimal | 应纳税所得额，单位：元 |
| taxAmount | BigDecimal | 应纳税额，单位：元 |
| taxRate | BigDecimal | 适用税率，例如 0.03 表示 3% |
| quickDeduction | BigDecimal | 速算扣除数，单位：元 |

失败时返回错误信息，说明第一个校验失败原因即可。

## 硬约束

- 必须本地可运行。
- 必须有自动化测试。
- 必须处理核心边界情况。
- 必须说明当前假设和限制。
- 使用 Java 8 语法。
- 使用 Spring Boot 2.7.x。
- 使用 Gradle 构建。
- 不引入数据库、Redis、MQ、登录、权限或复杂前端。

## 业务规则

### 计算公式

年度基本减除费用固定为 60000 元。

```text
应纳税所得额 = 年度工资收入 - 60000 - 年度专项扣除 - 年度其他扣除
```

如果计算结果小于或等于 0：

```text
应纳税所得额 = 0
应纳税额 = 0
税率 = 0
速算扣除数 = 0
```

如果应纳税所得额大于 0：

```text
应纳税额 = 应纳税所得额 * 适用税率 - 速算扣除数
```

### 税率表

| 级数 | 年度应纳税所得额 | 税率 | 速算扣除数 |
| --- | --- | --- | --- |
| 1 | 不超过 36000 元 | 3% | 0 |
| 2 | 超过 36000 元至 144000 元 | 10% | 2520 |
| 3 | 超过 144000 元至 300000 元 | 20% | 16920 |
| 4 | 超过 300000 元至 420000 元 | 25% | 31920 |
| 5 | 超过 420000 元至 660000 元 | 30% | 52920 |
| 6 | 超过 660000 元至 960000 元 | 35% | 85920 |
| 7 | 超过 960000 元 | 45% | 181920 |

税档判断规则：

1. 边界值包含在较低税档内，例如 36000 使用 3%，144000 使用 10%。
2. 超过边界 0.01 元时进入下一税档，例如 36000.01 使用 10%。

## 精度要求

- 金额计算使用 `BigDecimal`。
- 最终输出的 `taxableIncome`、`taxAmount`、`quickDeduction` 保留 2 位小数。
- 最终输出金额使用向下取整，即 `RoundingMode.DOWN`。
- `taxRate` 使用小数表示，保留 2 位小数，例如 3% 输出为 `0.03`。
- 中间计算不使用 `double` 或 `float`。

## 边界情况

至少覆盖以下情况：

1. 空请求体。
2. 缺少任一必填字段。
3. 任一字段为 `null`。
4. 任一字段为负数。
5. 任一字段不是合法数字。
6. 任一字段小数位超过 2 位。
7. 应纳税所得额小于 0。
8. 应纳税所得额等于 0。
9. 应纳税所得额正好等于税档边界值，例如 36000、144000、300000、420000、660000、960000。
10. 应纳税所得额刚超过税档边界值，例如 36000.01、144000.01。
11. 极大金额输入时不溢出、不使用科学计数法影响计算。

重复数据不是本 MVP 的特殊业务场景，因为本次不保存请求，也不做批量计算；相同输入重复调用时应返回相同结果。

## 暂不实现

1. 登录权限。
2. 数据库存储。
3. Redis / MQ。
4. 复杂前端页面。
5. 生产部署。
6. 复杂监控告警。
7. 批量计算。
8. 计算历史记录。
9. 专项附加扣除明细拆分。
10. 税收优惠、减免税、补税或退税流程。
11. 多地区、多年度政策配置。
12. 月度预扣预缴计算。

## 当前假设和限制

1. 所有输入金额均为人民币元。
2. 工资收入、专项扣除和其他扣除均按年度总额输入。
3. 年度基本减除费用固定为 60000 元，不做配置化。
4. 税率表使用当前文档中的年度综合所得税率表，不联网查询政策。
5. 本 MVP 只负责计算，不负责判断用户是否符合某项扣除资格。
