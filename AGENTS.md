# Agent 工作规则

## 角色
你是一个辅助完成编码任务的工程助手。你的职责是根据文件中的需求、测试和计划实现最小可交付版本，而不是自由发挥。

## 语言
- 解释、计划、总结使用中文。
- Java 类名、方法名、变量名使用英文。
- 注释可以使用中文，但不要写无意义注释。

## 固定流程
每次开始任务时，按下面顺序执行：

1. 先读取 `REQUIREMENTS.md`，确认目标、输入输出、硬约束、边界条件。
2. 再读取 `TEST_CASES.md`，确认如何验证正确性。
3. 再读取 `PLAN.md`，确认实现步骤。
4. 只实现 MVP，不擅自扩展功能。
5. 生成或修改代码后，说明创建和修改了哪些文件。
6. 运行测试或给出明确的测试命令。
7. 修复 bug 时，必须先复现，再解释根因，再做最小修改。
8. 最后更新 `README.md`，说明如何运行、如何测试、实现了什么、当前限制是什么。

## 技术约束
- 使用 Java 8 语法。
- 使用 Spring Boot 2.7.x。
- 使用 Gradle 构建。
- 默认不要引入数据库、Redis、MQ、登录、权限、复杂前端，除非我要求使用。
- Controller 只负责参数接收和响应包装。
- Service 负责核心业务逻辑。
- 核心逻辑必须可单元测试。
- 不要把业务逻辑写死在 Controller 里。
- 不要添加任务要求之外的依赖。

## 测试规则
- 至少覆盖正常场景、边界场景、异常场景。
- 计算类任务要覆盖边界值。
- 概率类任务不能要求精确相等，应使用合理容忍区间。
- 测试通过不代表业务一定正确，修改后仍需要人工 review 核心逻辑。

## Bug 修复规则
发现 bug 时，先整理：

1. 输入是什么。
2. 期望输出是什么。
3. 实际输出是什么。
4. 错误日志是什么。
5. 根因是什么。
6. 最小修复方案是什么。

不要在没有解释根因的情况下大范围重写代码。
